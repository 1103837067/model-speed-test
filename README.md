# LLM TPS Benchmark

纯前端 LLM 输出速度测试工具。直接在浏览器里对比不同模型的流式输出速度（字符/秒），所有数据存在本地，不经过任何后端。

## 为什么做这个

选模型时经常需要知道：哪个模型在我的网络环境下实际跑得最快？各家 API 的 TTFT（首 token 延迟）差多少？官方标的速度和实际体验差多远？

这个工具就是回答这些问题的——配好 API Key，选模型，点测试，结果自动保存成报告。

## 核心功能

- **多协议支持** — OpenAI 兼容、Google Gemini、Anthropic 三种 API 协议
- **自动发现模型** — 填好 Base URL 和 Key 后可一键拉取所有可用模型
- **批量速度测试** — 多选模型，支持并发和逐个两种测试策略
- **实时指标** — 测试中实时显示 TTFT、已输出字符数、当前速度、流式内容
- **测试报告** — 每次测试自动保存为独立报告，包含完整结果和排名，可随时回看
- **一键中断** — 随时终止所有正在进行的测试

## 注意事项

- **API Key 安全** — Key 存在浏览器 localStorage，这是纯前端工具，不会发往任何第三方服务器，但请注意不要在公共设备上使用后忘记清除
- **CORS 限制** — 浏览器直接调 API 需要目标服务支持 CORS。大部分 OpenAI 兼容中转服务都支持，但 Anthropic 官方 API (`api.anthropic.com`) 不支持浏览器直接调用，需走支持 CORS 的代理
- **并发测试的局限** — 浏览器对同一域名有并发连接数限制（通常 6 个），测试大量模型时建议用逐个模式或分批测试
- **速度受网络影响** — 测试结果反映的是「你的网络环境 → API 服务端」的端到端体验，不等于模型本身的推理速度

## 本地开发

```bash
pnpm install
pnpm dev
```

## 部署

已配置 GitHub Actions，推送到 `main` 分支自动部署到 GitHub Pages。

也可手动构建：

```bash
pnpm build
# 产物在 dist/ 目录
```
